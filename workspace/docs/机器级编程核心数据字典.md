# 机器级编程核心数据字典

这份字典整理了计算机底层世界（尤其是基于 x86-64 架构）中最核心的数据尺寸、Intel 历史术语以及与 C 语言的对应关系。在阅读汇编代码和进行底层 Debug 时，可随时查阅。

## 第一层：最底层的物理与进制基础

这是计算机世界的原子构成。

| 概念 | 英文 | 换算关系 | 核心意义 |
| :--- | :--- | :--- | :--- |
| **位** | **Bit** | = 0 或 1 | 硬件电路能存储的绝对最小单位（高/低电平）。 |
| **十六进制符** | **Hex Digit** | = 4 bits | 1个十六进制字符（0~F）。也就是常说的半字节(Nibble)。 |
| **字节** | **Byte** | = 8 bits = 2个十六进制符 | **CPU 内存寻址（编号）的最小单位！** Debug 时看到的一组代码（如 `A5`）就是一个字节。 |
| **缓存行** | **Cache Line**| 约 64 Bytes (典型值) | CPU 去物理内存拿数据的实际“起步进货量”。它不会只拿 1 字节，而是连带周围的 63 个字节一起打包拿走。 |

---

## 第二层：Intel 的“历史黑话”（核心必背！）

Intel 是从 16 位起家的，其词汇表被历史“锁死”。在汇编指令后缀（如 `movb`, `movl`, `movq`）中会频繁出现这些字母：

| Intel 术语 | 汇编后缀 | 包含字节数 | 包含位数 | 历史渊源 |
| :--- | :---: | :--- | :--- | :--- |
| **字节 (Byte)** | `b` | **1** Byte | 8 bits | 基础单位。 |
| **字 (Word)** | `w` | **2** Bytes | 16 bits | 因为 8086 是 16 位处理器，所以当时把 16 位定义为“一个字”。 |
| **双字 (Double Word)** | `l` (long) | **4** Bytes | 32 bits | 386 时代进化到 32 位，只能叫“双倍的字”。 |
| **四字 (Quad Word)** | `q` | **8** Bytes | **64 bits** | **我们现在 64 位机器的核心单位！** 也是指针和长整型的大小。 |

---

## 第三层：C 语言数据类型在 64 位系统(Linux)下的真实体积

高级语言里的变量到了底层实际占用的空间：

| C 语言类型 | 大小 (字节) | 对应的 Intel 汇编后缀 | 说明 |
| :--- | :---: | :---: | :--- |
| `char` | **1** | `b` | 刚好 1 个字节。 |
| `short` | **2** | `w` | 短整型。 |
| `int` | **4** | `l` | 标准整型。 |
| `long` | **8** | `q` | 长整型，在 64 位 Linux 中是 8 字节。 |
| **`char *` / `int *` 等指针** | **8** | `q` | **所有类型的指针统统是 8 字节！** 因为在 64 位系统下，内存地址的长度就是 64 位（8字节）。 |
| `float` | **4** | `s` (单精度) | 浮点数，使用特殊的浮点寄存器。 |
| `double` | **8** | `l` (双精度) | 双精度浮点数。 |

## 第四层：CPU 寄存器的“俄罗斯套娃”尺寸

以通用寄存器 `A` 为例，看看 CPU 是如何把一个 8 字节的寄存器切分给不同大小的 C 语言变量使用的：

| 寄存器名称 | 占用空间 | 对应的 C 数据类型 | 场景说明 |
| :--- | :--- | :--- | :--- |
| **`%rax`** | **8 字节** (64 bits) | `long`, `指针` | 整个大抽屉。操作指令通常是 `movq`。 |
| **`%eax`** | **4 字节** (32 bits) | `int` | 抽屉的低 4 字节。操作指令通常是 `movl`。 |
| **`%ax`** | **2 字节** (16 bits) | `short` | 抽屉的低 2 字节。操作指令通常是 `movw`。 |
| **`%al`** | **1 字节** (8 bits) | `char` | 抽屉的最底端 1 字节。操作指令通常是 `movb`。 |

*(注：如果在汇编里看到 `%eax`，通常说明它正在处理一个 32 位的 `int`；如果看到 `%rax`，通常说明它在处理一个 `long` 或者在搬运一个指针地址。)*

## 总结场景实战

假设你写了这样一段 C 代码：
```c
long a = 100;
char b = 'X';
int *p = NULL;
```
在底层的上帝视角看：
* `a` 是一个 **Quad Word (8字节)**，CPU 会用 `%rax` 这类 64 位寄存器，配合 `movq` 指令来搬运它。
* `b` 是一个 **Byte (1字节)**，它只占内存里 2 个十六进制符的空间，CPU 会用 `%al` 这类 8 位寄存器，配合 `movb` 来搬运它。
* `p` 是一个**地址（指针）**，哪怕它指向的是 4 字节的 int，它**自身作为地址**的长度依然是 **Quad Word (8字节)**，需要 `%rax` 和 `movq` 来处理！
