# CSAPP 第二章：信息的表示和处理 (Data Representation)

  - 第一部分：信息存储（底层基石与位操作）
    - 字节与十六进制
      - 快速在二进制、十进制和十六进制间转换
        - 为什么使用十六进制？
          - 二进制太长了，难看懂
          - 十进制和二进制很难转换，十六进制是完美的折中
      - 字长（Word Size）：32位与64位系统的区别
        - 最大可寻址内存
        - 关键区别在long类型和指针的大小
          - 指针大小和系统位数需保持一致，以便可以表示系统中任何可能得内存
    - 寻址与字节顺序
      - 大端法（Big Endian）：最高有效字节在前面
      - 小端法（Little Endian）：最低有效字节在前面（x86 常用）
      - 为什么选择小端法？（硬件友好）
        - 方便类型转换(截断)
        - 方便加法运算
    - 布尔代数基础
      - 基本运算：~ (NOT), & (AND), | (OR), ^ (XOR)
        - ~ 按位取反
        - & 常用来“提取”和“清除”某些特定的位
        - | 常用来强制“设置”某些位为 1，而不影响其它位
        - ^ 两个位不相同为1，相同为0
      - XOR的特性
        - 归零律 a ^ a = 0
        - 恒等律 a ^ 0 = a
        - 交换律和结合律 a ^ b = b ^ a, (a ^ b) ^ c = a ^ (b ^ c)
        - 经典应用
          - 不使用额外变量交换两个数 x = x ^ y; y = x ^ y; x = x ^ y;
      - 在物理电路上只有与非门和或非门，但是基于这俩组合实现了不同的ALU算数逻辑单元，又因为不同的ISA制定了很多指令，这些指令又基于ALU来实现。至于相同的ISA是怎么实现的，不同的cpu厂商有不同的实现方式
    - C语言中的位操作（Datalab 核心）
      - 位级运算：&, |, ~, ^（掩码、置位、清零）
      - 逻辑运算：&&, ||, !（与位运算的严格区分）
      - 移位运算
        - 左移（<<）：右边补 0
        - 右移（>>）：逻辑右移（无符号位，左补0） vs 算术右移（左补符号位）
  - 第二部分：整数表示（有符号与无符号）
    - 无符号数编码（Unsigned）
      - 基于传统二进制表示非负整数
    - 有符号数编码：补码（Two's Complement）
      - 核心思想：最高位（符号位）权重为负 ($-2^{w-1}$)
      - 范围不对称性：|TMin| = TMax + 1
    - 类型转换
      - 隐式与显式转换、
        - 隐式：有符号数会默认转换为无符号数
      - 规则：底层位模式不变，仅改变解释方式
    - 扩展与截断
      - 扩展：必须保持数值不变
        - 零扩展（无符号），高位全部补0
        - 符号扩展（有符号），高位全部补1
      - 截断：丢弃高位，可能改变数值与符号
  - 第三部分：整数运算（溢出与性能优化）
    - 加法与减法
      - 无符号加法与溢出检测
        - unsigned int x, y, sum; sum = x + y; if (sum < x) { //溢出 }
      - 补码加法与溢出检测（正溢出、负溢出）
        - 两个大正数相加，最后变成了负数
        - 两个大负数相加，最后变成了正数
      - 补码的非：-x 等价于 ~x + 1
        - 有符号数的相反数
      - 加法器ALU
        - CLA & CSA
    - 乘法与除法
      - 乘法截断：超出字长部分直接丢弃
    - 常数乘除优化（编译器技巧）
      - 乘 2 的幂：转换为左移 (x << k)
      - 除 2 的幂：转换为右移 (x >> k)
      - 负数除法向零舍入：加偏移量后算术右移
        - 硬件层面右移是直接向下舍入，但是C语言的设计是向0靠齐。所以C语言直接使用硬件的ALU能力是无法算出预期结果的，那么此时就在软件层面编译器通过加偏置量的方式来屏蔽这个差异
  - 第四部分：浮点数（IEEE 754 标准）
    - 二进制小数
      - 理解进位制本质：用 $2^{-n}$ 表示小数部分
      - 只能精确表示能够被拆解为 1/2,1/4,1/8,1/16 ... 的和的数字
      - 0.2在二进制小数：无限循环小数`0.001100110011...`
    - IEEE 浮点表示公式
      - $V = (-1)^s \times M \times 2^E$
      - 内存布局：符号位 (s) | 阶码 (exp) | 尾数 (frac)
        - 32位表示： [ 1位 ] [ 8位 ] [ 23位 ]
          - 127（即 $2^{k-1}-1$）是 IEEE 754 委员会在 8 位资源极其紧张的情况下，为了同时容纳“大数、小数、真零、无穷大、NaN”，并确保整体数学模型最优雅、最平滑，而计算出的那个“黄金分割点
        - 64位表示：[ 1 位 ] [ 11 位 ] [ 52 位 ]
    - 浮点数四种类别
      - 规格化：最常见，阶码非全0非全1（含隐藏的 1 和 Bias）
      - 非规格化：阶码全0，表示极接近 0 的数（逐渐下溢）
      - 特殊值：无穷大（阶码全1，尾数全0）
      - 非数 NaN（阶码全1，尾数非0）
    - 舍入规则
      - 向偶数舍入（Round-to-Even）：默认规则，避免统计偏差
    - C语言中的浮点运算
      - 缺乏结合律（精度丢失问题）
      - int, float, double 间的转换规则
